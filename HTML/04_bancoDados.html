Banco de Dados

Banco de Dados Relacionais

Um banco relacional é um tipo de banco de dados em que as estruturas de dados detém uma relação entre si. Em sua estrutura possuem:

  • Tabelas: estruturas que contém os dados
  • Chave Primária: coluna da tabela sem repetição de valores e que serve como identificador da tabela
  • Chave Estrangeira: é a chave primária que uma tabela recebe da outra para definir um relacionamento entre elas.

Modelagem de Banco de Dados

É a definição das regras de negócio do banco de dados a partir do escopo do projeto. É composto pelas seguintes etapas:

  1. MER e DER
  2. Modelo Lógico
  3. Modelo Físico

Modelo de Entidade e Relacionamento (MER)

É a descrição do BD em formato de entidades, atributos e relacionamentos. Para isso, precisamos dos seguintes conceitos:

  • Minimundo:
  • Abstração:

Entidades

São as tabelas de um Banco de Dados que representam atores e objetos únicos do mundo real. Elas possuem atributos que são suas características. Além disso, são classificadas em:

  • Forte:não possui chave estrangeira
  • Fraca:herda uma chave estrangeira
  • Entidade Associativa: entidades que associam duas entidades num relacionamento n para m

Atributos

São as características de uma entidade, que constituem as colunas das tabelas. Eles podem ser:

  • Atômicos:valores simples
  • Composto:com tipo diferentes de valores. Por exemplo, endereço.
  • Multivalorado: possuem mais de um valor. Por exemplo, telefone 1 e telefone 2.

Relacionamento

É a forma como as entidades se relacionam no sistema. Sobre ele estudamos:

  • Chave estrangeira:
  • Cardinalidade:define a relação de quantidade entre duas tabelas. Pode-se definir a quantidade máxima e mínima, entretando, o mais usual é demonstrar apenas a primeira.
    • (1:1): no máximo 1 entidade A se relaciona com no máximo 1 entidade B
    • (1:n): no máximo 1 entidade A se relaciona com no máximo várias(n) entidades B
    • (n:m): no máximo várias(n) entidade A se relaciona com no máximo várias(m) entidades B
