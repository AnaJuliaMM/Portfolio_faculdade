Banco de Dados Relacionais

Uma vez que um banco de dados é uma estrutura de armazenamento de dados, e o BD relacional é um tipo de banco composto por tabelas que se relacionam entre si por meio de uma coluna denominada chave estrangeira. Esse tipo de banco é caracterizado por:

  • Tabelas: estruturas que contém os dados
  • Chave Primária: coluna sem repetição de valores e que serve como identificador da tabela
  • Chave Estrangeira: é a chave primária que uma tabela recebe da outra para definir um relacionamento entre elas.

Modelagem de Banco de Dados

Esse é o processo de definição das regras de negócio de um banco de dados a partir do escopo do projeto. É composto pelas seguintes etapas:

  1. MER e DER
  2. Modelo Lógico
  3. Modelo Físico

1

MER e DER

Conceito Entidades Atributos Relação

Modelo de Entidade e Relacionamento (MER)

É a descrição textual do BD em formato de entidades, atributos e relacionamentos. Para isso, precisamos separar o problema em minimundos e realizar a abstração deles para identificar os elementos.

Diagrama de Entidade e Relacionamento(DER)

O DER é a representação gráfica do MER.

Entidades

São as tabelas de um Banco de Dados que representam atores e objetos únicos do mundo real. Elas possuem atributos que são suas características. Além disso, são classificadas:

  • Forte:não possui chave estrangeira
  • Fraca:herda uma chave estrangeira
  • Entidade Associativa: entidades que associam duas entidades num relacionamento n para m

Atributos

São as características de uma entidade, que constituem as colunas das tabelas. Eles podem ser:

  • Atômicos:valores simples
  • Composto:com tipo diferentes de valores. Por exemplo, endereço.
  • Multivalorado: possuem mais de um valor. Por exemplo, telefone 1 e telefone 2.

Relacionamento e Cardinalidade

É a forma como as entidades se relacionam no sistema. E a cardinalidade define a relação de quantidade no relacionamento entre duas tabelas. Pode-se definir a quantidade máxima e mínima, entretando, o mais usual é demonstrar apenas a primeira.

  • (1:1): no máximo 1 entidade A se relaciona com no máximo 1 entidade B
  • (1:n): no máximo 1 entidade A se relaciona com no máximo várias(n) entidades B
  • (n:m): no máximo várias(n) entidade A se relaciona com no máximo várias(m) entidades B

3

Modelo Lógico

Modelo Lógico

É uma representação do banco de dados mais próxima do modelo físico, representando as entidades como tabelas e os atributos como linhas. Nessa etapa são definidas os tipos de dados de cada coluna e suas restrições.

4

Modelo Físico

Modelo Físico

É a etapa de construção do Banco de Dados, que se dá através de duas ferramentas essenciais:

  • Sistema Gerenciador de Banco de Dados(SGBD)
  • Comandos em SQL

Em sala utilizamos o SGBD PostgreSQL e a interface PgAdmin.

5

Comandos em SQL

DDL

Comandos utilizados para criar, alterar e deletar as tabelas do BD. São eles:

  • CREATE TABLE
  • ALTER TABLE
  • DELETE TABLE

DML

Comandos utilizados para manipular os dados do BD, isto é, as linhas e campos das tabelas.

  • INSERT -inserir dados
  • UPDATE -atualizar dados
  • DELETE -deletar linha
  • SELECT -visualizar linha

DQL

Comandos utilizados para realizar buscas no BD.

  • WHERE -adiciona filtros nas buscas
  • SELECT -visualizar
  • FUNÇÕES -operações matemáticas, contagem, etc
  • ORDER BY -organizar a exibição
  • GROUP BY -organiza em função de uma coluna
  • ALIASES -apelidos
  • entre outros

Outros

DTL: Comandos de transação de dados

DCL: controle de segurança

6

Normalização

1FN

  • Atributos compostos viram atributos simples(atômicos)
  • Atributos multivalorados viram uma entidade

2FN

Separar as colunas que possuem dependências parciais e criar novas entidades

3FN

Remover as colunas com dependências transitivas e e criar novas entidades.
